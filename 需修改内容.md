# 需修改内容

本文档记录当前扫描器迭代分支在前后端联调完成后的后续修改点，优先围绕“扫描器迭代”职责展开。当前项目仍限定为本地授权靶场/课程项目使用，不面向第三方未授权目标。

## P0 必须优先完善

1. 任务与报告持久化
   - 现状：后端任务、报告、资产统计仍存放在内存中，服务重启后历史会丢失。
   - 修改目标：增加本地 SQLite 或 JSON 文件持久化，保存任务状态、扫描结果、报告索引、漏洞状态。
   - 影响页面：扫描管理、漏洞管理、报告中心、任务调度、侧边栏后端状态。

2. 设置持久化
   - 现状：系统设置页已经可以真实更新后端内存配置，但服务重启后恢复默认值。
   - 修改目标：将默认目标地址、源码路径、后端配置写入持久化配置文件。
   - 影响页面：系统设置、扫描管理。

3. 扫描任务状态恢复
   - 现状：页面刷新后可以读取后端任务，但后端重启后无法恢复历史任务。
   - 修改目标：后端启动时加载历史任务与最近报告，前端默认展示最近一次扫描结果。
   - 影响页面：扫描管理、漏洞管理、AI 修复、报告中心。

## P1 下一阶段重点

1. 通用爬取与路由发现
   - 现状：动态扫描主要针对固定路径，如 `/login`、`/comments`、`/admin`、`/profile/2`。
   - 修改目标：增加站点爬取、表单发现、参数发现，把扫描范围从固定靶场路径扩展为更通用的本地授权 Web 应用。
   - 注意：仍然只允许扫描本地或明确授权目标。

2. 漏洞去重与误报控制
   - 现状：扫描结果已能展示漏洞，但缺少统一去重、复核状态和置信度字段。
   - 修改目标：按漏洞类型、位置、参数、证据指纹合并重复结果，并增加 confidence 字段。
   - 影响页面：漏洞管理、AI 修复、报告中心。

3. 资产库
   - 现状：资产管理页展示的是扫描任务派生资产。
   - 修改目标：建立正式资产表，支持新增、编辑、选择资产发起扫描。
   - 影响页面：资产管理、扫描管理。

4. 调度能力
   - 现状：任务调度页只读取后端内存任务队列。
   - 修改目标：支持手动重跑、定时扫描、失败重试、任务取消。
   - 影响页面：任务调度、扫描管理。

## P2 可后续优化

1. 登录与权限
   - 现状：前端显示 Admin，但没有真实登录态和权限控制。
   - 修改目标：增加简单登录、角色、接口鉴权，避免所有人都能操作扫描任务。

2. 报告管理增强
   - 现状：报告可以通过后端接口打开 HTML/Markdown。
   - 修改目标：支持报告删除、重命名、下载文件、按风险等级筛选。

3. AI 修复建议增强
   - 现状：AI 建议接口已接入后端，并带本地中文模板兜底。
   - 修改目标：保存每次生成的建议版本，允许人工编辑、确认和导出。

4. 前端交互细节
   - 现状：核心页面已接真实接口，但部分页面仍以“后续平台能力”说明缺失功能。
   - 修改目标：随着后端接口补齐，逐步去掉说明性兜底文案，替换为真实操作。

## 当前已完成的联调基础

- 扫描管理可以调用后端启动扫描、轮询状态、展示结果。
- 漏洞管理、AI 修复、报告中心、资产管理、任务调度已优先读取后端数据。
- 顶部通知按钮已改为显示最近扫描事件。
- 顶部设置齿轮已删除。
- 扫描结果表已实现分页。
- 系统设置页按钮已能真实更新后端内存配置。
- 插件中心已删除。
- 侧边栏系统状态已改为后端状态。
